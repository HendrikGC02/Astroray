# pkg51 — Synthetic Telescope Observations

**Pillar:** 4
**Track:** B (post-process pass, self-contained)
**Status:** open
**Estimated effort:** 1–2 sessions (~4 h)
**Depends on:** pkg06 (pass registry), pkg47 (FITS loader, for PSF cubes)

---

## Goal

**Before:** Astroray renders physically accurate images, but they look
nothing like what a real telescope would see. There is no way to
simulate the effects of a telescope's point spread function (PSF),
detector noise, or finite exposure time.

**After:** A `TelescopeObservation` post-process pass convolves the
rendered image with a wavelength-dependent PSF (loaded from a FITS
cube generated by WebbPSF/STPSF) and adds a realistic detector noise
model (read noise + Poisson photon noise + dark current). The output
resembles a real telescope observation, enabling direct comparison
with actual data and realistic outreach imagery.

---

## Context

Synthetic observations are the bridge between simulation and data.
Astronomers routinely produce "what would JWST see?" images of their
simulations. Astroray's spectral pipeline makes this particularly
natural: each wavelength channel can be convolved with the correct
PSF for that wavelength, and the noise model can account for the
actual photon statistics.

This is not meant to be publication-grade instrument simulation (that
requires full detector modelling). It is "good enough for outreach and
rough predictions" — matching the scope defined in the astrophysics
design doc.

### PSF convolution

#### Wavelength-dependent PSF

The PSF varies with wavelength (diffraction limit scales as λ/D). A
PSF cube is a 3D FITS image with axes (x, y, wavelength). Each slice
is the PSF at one wavelength.

For Astroray's spectral pipeline, the convolution is applied per
spectral channel:

1. The renderer produces a multi-channel spectral image (or the pass
   receives the hero-wavelength-sampled result).
2. For each wavelength channel, select the closest PSF slice from the
   cube.
3. Convolve the channel image with the PSF slice via 2D FFT:

       I_convolved = FFT⁻¹[ FFT(I) · FFT(PSF) ]

4. Combine the convolved channels back into the output image.

If no PSF cube is available, the plugin falls back to a Gaussian PSF
with FWHM = 1.22 λ/D (diffraction limit), using user-specified
aperture diameter D.

#### Convolution implementation

2D FFT via a small built-in implementation (Cooley-Tukey radix-2, ~60
lines) or via FFTW if available as an optional dependency. Image and
PSF are zero-padded to the next power of 2 to avoid wrap-around
artifacts.

### Noise model

After PSF convolution, detector noise is added:

| Noise source | Model | Parameters |
|---|---|---|
| Photon noise | Poisson | λ = signal × gain × exposure_time |
| Read noise | Gaussian | σ = read_noise_e / gain |
| Dark current | Poisson | λ = dark_current × exposure_time |

All noise is applied in electron counts, then converted back to the
image's linear radiance units.

#### Parameters

| Parameter | Default | Description |
|---|---|---|
| `psf_file` | None | Path to PSF FITS cube. If None, uses Gaussian. |
| `aperture_diameter` | 6.5 m | Telescope aperture (for Gaussian fallback; JWST = 6.5 m). |
| `exposure_time` | 1000 s | Integration time. |
| `gain` | 1.0 e⁻/ADU | Detector gain. |
| `read_noise` | 6.0 e⁻ | Read noise per pixel per read. |
| `dark_current` | 0.005 e⁻/s | Dark current rate per pixel. |
| `n_reads` | 1 | Number of reads (noise averages as 1/√N). |
| `add_noise` | true | Enable/disable noise (for clean PSF-only mode). |
| `seed` | 0 | RNG seed for reproducible noise (0 = random). |

### Key design decisions

1. **Post-process pass, not per-ray.** PSF convolution is inherently
   an image-space operation. Applying it per-ray would be incorrect
   (the PSF describes the telescope optic's response to a point
   source, not individual photon paths). The pass runs on the finished
   rendered image.

2. **Pass ordering: after lensing, before denoising.** The telescope
   observation pass simulates what the telescope sees. Lensing (pkg50)
   happens in the universe; PSF + noise happen in the telescope.
   Denoising (OIDN) is a post-processing choice by the user. So the
   order is: render → lensing → telescope → (optionally) denoise.
   Note: denoising a noisy synthetic observation is physically
   questionable — the noise is the point. The user should generally
   disable OIDN when using this pass.

3. **Gaussian fallback.** Not everyone has WebbPSF installed or a PSF
   FITS cube available. The Gaussian diffraction-limited PSF is a
   reasonable approximation for many purposes and makes the pass
   usable out of the box.

4. **Built-in FFT, optional FFTW.** A simple radix-2 FFT is adequate
   for image sizes up to ~4K×4K. FFTW is faster but adds a dependency.
   The built-in version keeps the plugin self-contained; FFTW can be
   added as an optional accelerator later.

5. **Noise in electron counts.** The noise model operates in the
   detector's natural units (electrons), not in the image's radiance
   units. This ensures Poisson statistics are applied correctly
   (Poisson noise on photon counts, not on arbitrary intensity values).
   The conversion factor is gain × exposure_time × solid_angle, where
   solid_angle is derived from the pixel scale.

## Acceptance criteria

- [ ] `TelescopeObservation` registered via
      `ASTRORAY_REGISTER_PASS("telescope", TelescopeObservation)`.
- [ ] Gaussian PSF mode: a point source renders as a smooth Airy-like
      disk with FWHM consistent with 1.22 λ/D for the specified
      aperture.
- [ ] FITS PSF cube mode: convolution with a loaded PSF cube produces
      a visually different (more realistic) result than the Gaussian
      fallback.
- [ ] Noise model: at short exposure times, visible grain; at long
      exposure times, smooth image. SNR scales as √(exposure_time) for
      photon-noise-dominated regime.
- [ ] Reproducibility: same seed produces identical noise.
- [ ] Clean mode: `add_noise=false` produces a noise-free convolved
      image.
- [ ] No-PSF mode: if both `psf_file` and `aperture_diameter` are
      unset, the pass adds noise only (no convolution).
- [ ] Blender addon exposes all telescope parameters.
- [ ] All existing tests pass.
- [ ] ≥8 new tests covering: Gaussian PSF shape, FITS PSF loading,
      convolution correctness (delta function → PSF), noise statistics,
      SNR scaling, reproducibility, clean mode, pass ordering.

---

## Non-goals

- Do not implement full instrument simulation (detector non-linearity,
  cosmic rays, flat fielding, bias subtraction). This is outreach-
  grade, not calibration-grade.
- Do not implement multi-filter compositing (combining F115W + F200W
  into an RGB image). The user can render at different wavelengths and
  composite externally.
- Do not implement adaptive optics or atmospheric seeing (ground-based
  telescopes). Space-based diffraction-limited PSFs only.
- Do not implement coronagraphic modes.
- Do not require WebbPSF as a build or test dependency.
