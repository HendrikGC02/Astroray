# Astroray — HDRI Environment Map Implementation Rules

## Project Context
C++ path tracer (C++17) with pybind11 Python bindings for Blender integration.
Header-only renderer: `include/raytracer.h` and `include/advanced_features.h`.
Standalone binary: `apps/main.cpp`. Python module: `module/blender_module.cpp`.
Blender addon: `blender_addon/__init__.py`.

## Current Task
Add HDRI environment map support with importance sampling and MIS-weighted NEE.
See `docs/HDRI_IMPLEMENTATION_PLAN.md` for the full task breakdown.

## Critical Rules

### Architecture
- Everything in `include/raytracer.h` is header-only. The EnvironmentMap class goes here too.
- `#define STB_IMAGE_IMPLEMENTATION` goes in ONE .cpp file only (`src/stb_impl.cpp`), NEVER in headers.
- The `luminance()` function already exists in raytracer.h — use it, do not redefine.
- `Vec3` default-constructs to (0,0,0). `Ray` auto-normalizes its direction.

### Rendering Conventions (CRITICAL — violating these causes bugs)
- `Material::eval(rec, wo, wi)` returns **brdf × NdotL** (cosine INCLUDED). Do NOT multiply by NdotL again.
- `sampleDirect()` returns the full combined NEE+MIS estimate. Do NOT multiply by NdotL in the caller.
- Gamma correction is applied ONCE inside `Renderer::render()`. Do NOT apply gamma to HDR data.
- The existing firefly clamp (luminance > 20) in the render loop handles HDR hot pixels.
- `wasSpecular` flag controls emission double-counting. Environment map must follow the same pattern:
  camera rays and specular bounces add environment directly; diffuse bounces use NEE only.

### Code Style
- Use `std::clamp`, `std::max`, `std::min` (C++17)
- No `using namespace std;` in headers
- Use `M_PI` for pi (already used throughout)
- Use `float` for all rendering math (the GR integrator will use double later, not now)
- Initialize ALL struct members — `BSDFSample` has no default init for `pdf` or `isDelta`
- Use `1e-6f` or `0.001f` for epsilon guards, matching existing code

### Coordinate System
- Y is up (matches Blender)
- Equirectangular mapping: θ (polar) measured from +Y axis, φ (azimuthal) in XZ plane
- `direction.y = 1` → top of env map (v = 0), `direction.y = -1` → bottom (v = 1)

### Testing
- Tests use `pytest` with assertions (NOT `return True/False`)
- Test images saved to `test_results/` (gitignored)
- Use `from base_helpers import create_renderer, setup_camera, render_image, save_image`
- Standard test resolution: W=200, H=150. Samples: SAMPLES_FAST=16 for quick tests

### Anti-patterns (DO NOT)
- Do NOT create a giant emissive sphere for environment lighting (the old hack we're replacing)
- Do NOT use `new`/`delete` — use `std::shared_ptr` and `std::make_shared`
- Do NOT add external dependencies beyond stb_image.h (already in include/)
- Do NOT modify the per-pixel render loop structure (only the background lookup inside pathTrace)
- Do NOT change `LightList::sample()` or `LightList::pdfValue()` — environment map sampling is separate
